## khulnasoft-init

Initialize configurations for Khulnasoft local development.

A `khulnasoft/config.toml` file is created in your current working directory. This configuration is specific to each local project.

> You may override the directory path by specifying the `KHULNASOFT_WORKDIR` environment variable or `--workdir` flag.

In addition to `config.toml`, the `khulnasoft` directory may also contain other Khulnasoft objects, such as `migrations`, `functions`, `tests`, etc.
